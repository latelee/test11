hexo -i 192.168.31.225

解决：
在根_config.yml的git仓库，加上用户名。如
https://<用户名>@github.com/<用户名>/<用户名>.github.io
如：
https://lisanch@github.com/lisanch/lisanch.github.io

修改提交者信息
hexo d时，无须再次输入提交者信息，因为git全局已经设置过了。当执行此操作后，在.deploy_git目录下有.gi，修改里面的config文件。在后面添加：
[user]
	name = Late Lee
	email = li@latelee.org

## 页面
### 添加订阅：
npm install hexo-generator-feed --save

在根_config.yml添加：
feed:
  type: atom
  path: atom.xml
  limit: 30

### 关于页面
hexo new page about
生成source/about/index.md

### 图片
设置根目录_config.yml的post_asset_folder为true。
在md文件同级目录创建同名目录，放图片，在md文件使用图片：
{% asset_img 图片文件+后缀（不用写目录） [这是描述，可不写] %}
比如：
{% asset_img IMG_20140329_120348.jpg %}

## 评论，来必力
https://livere.com/
注册、登陆（有些不稳定，登陆几次）、安装代码，代码片段中带有data-uid，复制之，到主题的_config.yml中livere后面。评论时要注册来必力账户，体验有些不好。

## 导入wordpress文章
先在wordpress后台导出xml文件，安装插件：
npm install hexo-migrator-wordpress --save

hexo migrate wordpress wordpress_back.xml
